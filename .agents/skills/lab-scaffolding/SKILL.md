# Lab Scaffolding

이 스킬은 실습 과제가 너무 어렵거나 너무 쉬워지지 않도록 조절하는 기준을 제공한다.

## 5단계 실습 피라미드

1. 따라하기
2. 가이드 실습
3. 챌린지
4. 미니 프로젝트
5. 캡스톤

가능하면 레슨 단위에서는 1~3단계, 모듈 단위에서는 3~4단계, 과정 마지막에서는 5단계를 사용한다.

## 스타터 코드 원칙

- 처음부터 실행 가능해야 한다.
- 학습자가 작성할 부분은 `TODO`로 분리한다.
- 학습목표와 무관한 보일러플레이트는 미리 제공한다.
- 과정 후반으로 갈수록 제공 코드를 줄인다.

## 힌트 시스템

힌트는 한 번에 정답을 주지 않는다.

1. 방향 제시
2. 접근법 제시
3. 코드 스켈레톤 제시
4. 필요하면 예시 솔루션 공개

## 캡스톤 설계 기준

- 실제로 설명 가능한 문제를 다룬다.
- 2~4주 또는 과정 규모에 맞는 현실적인 범위로 잡는다.
- 핵심 학습목표를 통합적으로 검증한다.
- 필수 기능과 선택 기능을 나눈다.
- 포트폴리오 가치가 있어야 한다.

## 환경 설계

- 도구, 계정, 데이터, 버전, 하드웨어 요구사항을 문서에 남긴다.
- 환경이 불분명하면 접근성이 높은 무료 도구를 기본안으로 제시한다.
- 자동 검증 방법이 있으면 과제와 함께 제시한다.
